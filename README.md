# VK ↔ Telegram Bridge

Мост между сообщениями сообщества **ВКонтакте** и **Telegram**.

Сообщения, которые люди пишут в сообщество ВК, пересылаются вам в Telegram в виде:

```
👤 Имя Фамилия (vk.com/id123456): текст сообщения
```

Вы отвечаете прямо в Telegram — и ответ уходит обратно нужному собеседнику в ВК.

## Возможности

- **Отображение отправителя** — имя, фамилия и ссылка на профиль у каждого входящего.
- **Выбор получателя** через inline-кнопки (`/write`) — список всех, кто когда-либо писал.
- **Ответ через reply** — сделайте reply на сообщение, и бот сам определит получателя в ВК.
- **Проброс медиа в обе стороны**: фото, видео (или ссылка, если файл большой), голосовые,
  стикеры (ВК → картинка, Telegram `.webp` → `.png` в ВК), документы, GIF.
- **Контекст ответа** — если отвечаешь reply на сообщение, оно целиком переотправляется
  отдельным сообщением с пометкой ↩️ (с вложениями и стикером, без кавычек и
  без вложенных ответов), в обе стороны.
- **Показ правок** — отредактированное сообщение приходит с пометкой ✏️ (VK → TG и TG → VK).
- **Статус «печатает»** — печатает в ВК → видно в Telegram; пишешь в Telegram →
  собеседник в ВК видит «печатает…».
- **Красивое имя отправителя** — кликабельное имя-ссылка на профиль ВКонтакте.
- **Режим пакетной отправки** — копите несколько сообщений и отправляйте их разом.
- **Локальное хранилище SQLite** — соответствие `vk_user_id ↔ имя`, последний собеседник,
  привязка `message_id` для корректного reply.

## Команды бота

| Команда / кнопка        | Действие                                                        |
|-------------------------|-----------------------------------------------------------------|
| `/start`                | Приветствие и кнопка выбора собеседника                         |
| `/write`                | Показать список собеседников (inline-кнопки)                    |
| `/who`                  | Показать, кому ты сейчас пишешь                                 |
| `/cancel`               | Выйти из любого режима (выбор получателя / составление)         |
| reply на сообщение      | Отправить ответ конкретному собеседнику без кнопок             |
| 📨 Отправить всё        | Отправить накопленные сообщения в ВК (с задержкой 0.5 c)        |
| 🗑 Очистить             | Очистить очередь без отправки                                   |
| 👁 Показать очередь     | Показать накопленные сообщения                                  |
| 🔚 Завершить диалог     | Выйти из переписки с собеседником (сообщения больше не уходят)  |

Имя текущего получателя показывается в подсказке поля ввода — видно, кому пишешь.
При ответе через reply бот переотправляет сообщение, на которое ты ответил, отдельным
сообщением с пометкой ↩️ (без кавычек, без вложенных ответов).

## 1. Создание сообщества ВК и получение токена

1. Создайте сообщество: [vk.com/groups](https://vk.com/groups) → **Создать сообщество**.
2. Откройте **Управление** → **Настройки** → **Работа с API** → **Ключи доступа** →
   **Создать ключ**. Обязательно отметьте права **Сообщения сообщества** И
   **Управление сообществом** — последнее нужно для Long Poll (без него VK вернёт
   ошибку `[15] Access denied`). Скопируйте полученный токен — это `VK_TOKEN`.
3. Включите приём сообщений: **Управление** → **Сообщения** → **Сообщения сообщества: Включены**.
4. Включите **Long Poll API**: **Управление** → **Работа с API** → **Long Poll API** →
   состояние **Включено**, версия API — последняя (например `5.199`).
   В разделе **Типы событий** включите **Входящее сообщение**.

## 2. Создание Telegram-бота

1. Напишите [@BotFather](https://t.me/BotFather) команду `/newbot`, задайте имя и username.
2. Скопируйте выданный токен — это `TG_BOT_TOKEN`.
3. Узнайте свой Telegram ID у [@userinfobot](https://t.me/userinfobot) — это `TG_OWNER_ID`
   (бот будет общаться только с этим пользователем).

## 3. Настройка `.env`

Скопируйте пример и впишите свои значения:

```bash
cp .env.example .env
```

```dotenv
VK_TOKEN=токен_сообщества_вк
TG_BOT_TOKEN=токен_телеграм_бота
TG_OWNER_ID=твой_telegram_id
```

## Запуск локально (через uv)

Требуется [uv](https://docs.astral.sh/uv/) и Python 3.14+.

```bash
uv sync            # установить зависимости из uv.lock
uv run python main.py
```

Запуск тестов:

```bash
uv run pytest
```

## Запуск через Docker

```bash
docker compose up -d        # собрать и запустить в фоне
docker compose logs -f      # смотреть логи
docker compose down         # остановить
```

База данных хранится в Docker volume `bridge_data` (`/data/bridge.db` внутри контейнера),
поэтому история собеседников не теряется при перезапуске.

> ⚠️ После изменения кода пересобирайте образ: `docker compose up -d --build`.
> Просто `docker compose up -d` берёт старый закешированный образ.

## Частые ошибки

| В логах                                   | Причина и решение                                                                 |
|-------------------------------------------|-----------------------------------------------------------------------------------|
| `VK API [15] Access denied ... scopes`    | У токена нет права **Управление сообществом**. Пересоздай ключ доступа с этим правом. |
| Ошибка Long Poll, событий не приходит      | Включи **Long Poll API** и тип события **Входящее сообщение** в настройках сообщества. |
| Бот молчит в Telegram                      | Проверь `TG_OWNER_ID` — бот отвечает только этому пользователю.                    |

## Структура проекта

```
vk_bridge/
├── main.py            # точка входа, asyncio.gather() для VK и TG
├── vk_listener.py     # VkGateway: Longpoll, имена, отправка/загрузка в ВК
├── tg_handler.py      # роутеры aiogram, FSM, режим пакетной отправки
├── database.py        # работа с aiosqlite
├── media.py           # конвертация и проброс медиа в обе стороны
├── config.py          # pydantic-settings
├── tests/             # pytest
├── Dockerfile
├── docker-compose.yml
├── .dockerignore
├── .env.example
└── pyproject.toml     # управляется через uv
```

## Технологии

Python 3.14+ · aiogram v3 · vk_api (Longpoll) · aiohttp · aiosqlite · Pillow · pydantic-settings.
